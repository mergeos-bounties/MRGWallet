# Firefox temporary add-on load

MRGWallet ships as a **temporary add-on** for Firefox development.  
Permanent distribution via Mozilla Add-on Store is tracked in a future bounty.

## Build

```bash
npm run build:extensions
```

Output: `dist/extensions/firefox/manifest.json`

## Load in Firefox

### Desktop

1. Open `about:debugging` in the address bar
2. Click **This Firefox**
3. Click **Load Temporary Add-on…**
4. Select `dist/extensions/firefox/manifest.json`
5. The MRGWallet icon appears in the toolbar — click to open

### Android (Firefox Nightly)

1. Install **Firefox Nightly** from Google Play
2. Enable **Remote debugging via USB** (`about:config` → `devtools.debugger.remote-enabled` → `true`)
3. Connect device via USB, open `about:debugging` on desktop → **This Firefox** → **USB devices**
4. Click **Load Temporary Add-on** and pick `manifest.json`

## Extension ID

The manifest declares:

```json
"browser_specific_settings": {
  "gecko": {
    "id": "mrgwallet@mergeos.shop",
    "strict_min_version": "109.0"
  }
}
```

The `id` is required by Firefox for persistent storage. Chrome ignores `browser_specific_settings`.

## Manifest differences from Chrome

| Key | Chrome | Firefox |
|-----|--------|---------|
| Manifest version | MV3 | MV3 |
| Background | Service worker (`background.js`) | Same (Firefox MV3 supports service workers) |
| Extension ID | Generated by Chrome | Explicit via `browser_specific_settings.gecko.id` |
| Permissions | `storage`, `alarms` | Same |
| Host permissions | `mergeos.shop`, `scan.mergeos.shop` | Same |

## Troubleshooting

- **"Add-on is not compatible"** — `strict_min_version` requires Firefox 109+. Update if testing on older ESR.
- **Popup not opening** — Check `about:debugging` → Inspect → Console for errors. The popup HTML is at `web/index.html` relative to the extension root.
- **Storage cleared on restart** — Temporary add-ons lose storage when Firefox restarts. This is expected for development.
